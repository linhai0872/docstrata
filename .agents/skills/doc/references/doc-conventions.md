# Doc Conventions — 通用文档规范

四层文档共用的写作规范，来自对 agent 友好文档的调研（AWS RAG best practices、llms.txt 标准、Anthropic Agent Skills）。目标：人机双消费，人读得懂、agent 检索得准。

## 结构

- **每节带摘要**：每个二级/三级标题下，首句给出该节的一句话摘要。改善 RAG 检索锚点。
- **上下文自包含**：每份文档能独立被理解。定义缩写、明确项目特定术语，不假设读者有外部背景。防止 LLM 幻觉补全。
- **小而聚焦**：一个文档一个主题。过大就拆，宁可多文件互链。

## 格式

- **Markdown 优先**：纯 Markdown，去 HTML 噪声。这是 LLM 的原生格式。
- **避免大表格**：表格的二维关系 LLM 线性读取时易丢失。简单对照可用表格，复杂结构改用多级列表。
- **代码/路径**：引用文件用 `path:line` 形式（可点击）。但描述"能力/领域概念"优先于写死文件路径（路径随重构过期，概念稳定）。

## 措辞与语气

文档面向人与 agent 双读者，概述/背景/结论这类 prose 段落最容易出 AI 腔。本节约束怎么说，completeness contract 决定说什么：措辞简洁不删信息。

- **结论先行**：每节、每个决策条目先给结论，再补支撑。与"每节带摘要"同源。
- **正面陈述**：直接说"X 是 Y"，少用"不是…而是…"的对立框架。真要区分两面，就并列两个正面分句。（例外：逻辑充要条件、技术定义里 load-bearing 的边界澄清，如"INDEX.md 不是 Working Memory 层"）
- **不预告总结**：有结论直接当末句说，去掉"综上所述""一句话总结""简而言之"。
- **杀 filler**：删"值得注意的是""首先我们需要""让我们一起来看看""其实"。
- **不复述**：一个点讲清一次，不用"翻成人话""换句话说"重说一遍。
- **慎用破折号**：解释、列举优先用冒号或独立句子，破折号留给真正的强调，别当万能连接符。

不迁移的对话规则（套到文档反而有害）：句数上限、pros/cons 点数上限、按问题复杂度缩水。文档篇幅由 completeness contract 决定。

## 元数据

每份文档头部带 `last-verified: YYYY-MM-DD`，尾部维护 `## 变更记录`：
```
## 变更记录
- 2026-06-03 首次生成
- 2026-06-10 更新 {section}（{原因}）
```
首次与增量都追加一行（日期 + 改了哪些 section + 一句话原因）。详见 methodology.md 的 STAMP。

超期或低置信内容主动标注 `[推断]` / `[待确认]`，不要伪装成确定事实。

## 层间链接

- 下游层引用上游层用相对链接：dev → wiki/knowledge/requirements。
- 引用 knowledge 层时引摘要条目，不复制原始材料全文。
- 避免循环引用。
