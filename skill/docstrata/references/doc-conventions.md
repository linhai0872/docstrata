# Doc Conventions — 通用文档规范

各层文档共用的写作规范。目标：人机双消费——人读得懂，agent 检索得准，文风不像 AI 生成。

来源：agent 友好文档调研（AWS RAG best practices、llms.txt、Anthropic Agent Skills）+ 中文 anti-slop 实践（deslop-zh、stop-slop-cn、natural-chinese-protocol、talk-normal 等）。

## 结构

- **每节带摘要**：每个二级/三级标题下，首句给出该节的一句话摘要。改善 RAG 检索锚点。
- **上下文自包含**：每份文档能独立被理解。定义缩写、明确项目特定术语，不假设读者有外部背景。防止 LLM 幻觉补全。
- **小而聚焦**：一个文档一个主题。过大就拆，宁可多文件互链。

## 格式

- **Markdown 优先**：纯 Markdown，去 HTML 噪声。这是 LLM 的原生格式。
- **避免大表格**：表格的二维关系 LLM 线性读取时易丢失。简单对照可用表格，复杂结构改用多级列表。
- **代码/路径**：引用文件用 `path:line` 形式（可点击）。但描述"能力/领域概念"优先于写死文件路径（路径随重构过期，概念稳定）。

## 可读性：先修关系，再修表达

「读起来累」通常是两个病叠加，分开治。

### 病一（主因）：孤岛条目 → 补向上指针

读一条 dev 结论却找不到它解决的是哪个需求，理解成本陡增。修法：**条目可选带一句向上指针**，链到上游需求或 PRD 意图——`（为满足 requirements 的 X 共识）` 或 `[为解决 prd 的 Y]`。有上游就带，没有不强求。关系链补上，大半的「累」自己就消了。

### 病二（次因）：表达太密 → 两段式

句子拧太紧（箭头链、嵌套括号、slug 堆叠）让人逐字解压。修法：**两段式**，density 不删只分层。

- **第一句**：自然语言下结论，带上下文。人读这句就懂大概。
- **下面**：密集证据、配置值、命令、路径。agent 往深读。

示例：`合并带依赖变更的上游后容易白屏，记两个收尾坑 → ① bun install ② 用 dev:vtrust 验证`

### 可读性预算（硬约束）

- 一句话一个意思。连续 `→` 箭头链 ≤ 2 跳，更长拆列表或分句。
- 括号嵌套 ≤ 1 层。第二层补充另起一句。
- 项目专属缩写/slug 首次出现给一次展开，别假设读者已知。

## 措辞与语气

prose 段落（概述/背景/结论）最容易出 AI 腔。completeness contract 决定说什么，本节约束怎么说。

### 句法层

- **结论先行**：先结论，再补支撑。每节首句 = 摘要 = 两段式第一句，一脉相承。
- **正面陈述**：直接说"X 是 Y"。少用"不是…而是…"——真要区分，并列两个正面分句。例外：技术定义里必须划边界的地方（"INDEX.md 不是 Working Memory 层"）。
- **动词直接上**：拆掉"进行/实施/实现 + 名词"壳。`对系统进行测试` → `测试系统`；`实现性能的提升` → `提升性能`。
- **主语明确**：不躲被动，不滥用"它"。`建议下一阶段重构` → `后端团队计划 Sprint 12 重构支付模块`。
- **句子长短交错**：短句下判断（≤15 字），长句补原因（20–40 字）。全篇等长句是 AI 文本的统计特征。

### 用词层

- **说具体**：数据/证据替代空泛形容。`性能显著提升` → `P95 延迟从 320ms 降至 85ms`。
- **杀 filler**：删 `值得注意的是` / `首先我们需要` / `其实` / `显而易见` / `众所周知`。
- **禁互联网黑话**：`赋能` / `抓手` / `闭环` / `打通` / `颗粒度` / `沉淀`（非技术义）。语义模糊，agent 无法提取可操作信息。
- **声明式可判定**：主张写成能裁决的句子，避免"应该/尽量/灵活"。
- **认知标注替代限定语**：有 `[推断]` 标就够了，不再堆"可能在一定程度上或许"。

### 结构层

- **不预告总结**：有结论直接当末句，去掉"综上所述""简而言之"。
- **不复述**：一个点讲清一次，不用"换句话说"重说一遍。
- **结构为内容服务**：有内容才写 section，没内容不补空话凑三段式。
- **慎用破折号**：解释、列举用冒号或独立句，破折号只留给真正的强调。

### 保真层

- **代码字面量不动**：标识符、命令、API 路径、JSON 键名、版本号、错误码——原样保留。改动字面量会破坏 agent 代码搜索。
- **英文专有名词不译**：不翻译、不意译、不替换。

### 不迁移的规则

对话类约束（句数上限、按复杂度缩水）和社媒口语风格不套进文档。文档篇幅由 completeness contract 决定。

## 元数据

每份文档头部带 `last-verified: YYYY-MM-DD`，尾部维护 `## 变更记录`：
```
## 变更记录
- 2026-06-03 首次生成
- 2026-06-10 更新 {section}（{原因}）
```
首次与增量都追加一行（日期 + 改了哪些 section + 一句话原因）。详见 methodology.md 的 STAMP。

超期或低置信内容主动标注 `[推断]` / `[待确认]`，不要伪装成确定事实。

## 层间链接

- 下游层引用上游层用相对链接：dev → wiki/knowledge/requirements/prd。
- 重叠内容以上游为源、下游引用不复制：wiki/requirements 的「定位/用户/价值」引 prd；引用 knowledge 层引摘要条目，不复制原始材料全文。
- 避免循环引用。
