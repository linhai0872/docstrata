# Compact — 长期记忆收缩操作

把随迭代膨胀的层文档收缩回有界、密集、可检索的状态。手动触发 `/docstrata compact`，无自动催。

## 为什么需要

五步循环只有「增量更新」，只增不减。长期项目的层文档必然膨胀——dev 按日期 append 工程实录，requirements 决策记录无限堆叠。单一 markdown + agent 自由写回会迅速变乱、自相矛盾，agent 静默丢指令、重复已做过的工作，这有实证。compact 补上增量更新缺的另一半：定期把日志蒸馏成知识、把过期折叠进归档。

## 何时触发

纯手动。用户跑 `/docstrata compact` 或 `/docstrata compact dev` 才执行。不主动提示、不自动跑。

STAMP 时顺手在 `last-verified` 行标注当前行数（如 `last-verified: 2026-06-24 | dev.md 312 行`），给用户被动的体量信号，要不要 compact 由用户判断。这是数字，不是催促。

## 核心操作：先压，再拆，归档不删

compact 分三步，前一步能解决问题就不进下一步：

1. **压（蒸馏 + 去重 + 归并）**
   - **丢可复原项**：能从 code + git log + release notes 复原的内容删掉。判据见下「毕业门槛」。
   - **去重**：同一事实散在多处，合并到一处，其余删。
   - **按主题归并**：散在不同日期块的同主题条目，折进同一主题节。`按日期堆 = 无限增长，按主题归并 = 有界`——这是 compact 最核心的动作。
   - 压完若已回到合理体量（如 < 300 行），停，不进第 2 步。

2. **拆（仅当压完仍超限）**
   - 单纯拆文件只是把臃肿挪个地方，所以**先压再拆**。压完仍超 300 行才按主题拆成子文件，主文件留索引指向子文件。

3. **归档不删**
   - 被压掉的时间线日志/被取代的决策，不物理删除——移到 `docs/archive/`（按里程碑或日期命名），需要时可 rehydrate。git 历史本就保留全部，archive 是给人/agent 的可读冷存。

## 毕业门槛（什么留、什么丢）

留与丢的分界线是 code + git log 能否复原。能复原的删，不能复原的留。三类该留：

- **坑**：不知道就会重踩的（如「合并带依赖变更的上游后必须先 `bun install`，否则白屏」）。
- **被否的方案 + 原因**：试过放弃了的，防后人重蹈。
- **非显然的决策理由**：代码里看不出的「为什么这么选」。

该丢的内容 git 那边有，未来 agent 不知道也不会犯错：

- 叙事/统计（「合并 69 commits 零冲突」「继承了哪些能力」）→ git log / release notes。
- 一次性操作流水 → commit message / PR。
- **working memory 泄漏物**：`现状快照`、`本期进度`这类在途易失状态——它们属于运行时，不该 append 进长期记忆；compact 时直接踢出。

## 蒸馏的保真要求（防总结幻觉）

压缩时叙事可以抽象，硬事实必须逐字保留。

abstractive 总结会把硬事实总结错（faithfulness 幻觉）。compact 用**叙事抽象 + 硬事实逐字保留**的混合范式（同 source-criticism 的事实/推断标注）：

- **叙事可压**：背景、过程、动机可以提炼成一两句。
- **硬事实逐字保留**：配置值、错误码、版本号、命令、裁决结论——逐字钉住，不改写。缺则标 N/A，不编造。
- 压缩后认知标注不丢：`[事实]` / `[推断]` / `[实测]` 跟着内容走，不因为变短就抹掉来源。

## 跨层规则（各层 compact 重点不同）

- **dev**：最易膨胀。重点是丢可复原叙事 + 按主题归并日期块。退化成 devlog 时收得最狠。
- **requirements**：决策记录只增不改写（事实留痕），但**被取代的旧决策折叠**——保留结论 + supersede 指针（如「D16 早期单模型方案，已被下文双模型取代」），不展开旧细节。
- **prd**：本就该重写而非追加，膨胀风险低；compact 时检查是否有该上移到 requirements 的历史、或该下沉到 roadmap archive 的过期条目。
- **wiki / knowledge**：描述/索引层，膨胀慢；compact 主要去重 + 删过期描述。

## 编号当交叉引用 handle（保留）

决策若用了编号（D1、D23 这种），编号是项目里实际流通的交叉引用 handle（commit、对话、其他文档都在引）。compact 折叠决策时**保留编号**，删了会断引用。去掉的是「ADR」这类术语黑话，不是编号本身。

## review-all 协议（借 Cursor Memory Bank）

compact 动手前必须通读目标文件全文——只扫局部会漏掉跨段重复和该归并的同主题条目，而这恰是膨胀的主因。

compact 单层就读单层全文；`/docstrata compact`（全部）就逐层通读。即使大部分段落不需要改，也先全文读完再动手。
