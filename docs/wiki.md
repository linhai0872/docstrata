# doc-layers

> 一个 Claude Code Skill，按四层知识结构为任意项目生成和维护文档。

last-verified: 2026-06-03

## 这是什么

一套文档生成工具。给它一个项目目录，它能产出四种不同性质的文档：业务全景、需求共识、业务知识库、开发结论。每种文档对应不同受众和用途。

## 解决什么问题

项目形态各异（全栈服务、Dify 应用、CLI/MCP、Skill），文档需求却是共通的：业务要看懂系统、需求要留痕、业务材料要可检索、开发结论要沉淀。市面没有单一工具覆盖这四类，doc-layers 用一个 skill 的四个子命令解决。

## 谁在用

开发团队。后续有价值会开源给更广的社区。

## 核心功能

- **wiki** — 生成业务全景文档，所有人（含非技术）能快速理解系统。
- **requirements** — 还原/固化需求共识与开发计划（ADR 性质）。
- **knowledge** — 把业务原始材料整理成可检索的索引。
- **dev** — 记录开发推断与实践结论。
- **all** — 按依赖顺序一次生成全部。

特点：工具先自动探索项目上下文，只在信息有缺口时才向人提问（结对 grill）；增量更新，保留人工修改。

## 关键概念

- **CoALA 四层** — Working / Episodic / Semantic / Procedural，本工具四层文档的理论来源。
- **Completeness Contract** — 每层声明的信息维度集合，驱动"该不该问人"。
- **Gap-Driven Grill** — 只对探索不到的信息维度提问，把人当作补全 context 的工具。

## 如何运转（概览）

每层都走同一个循环：探索上下文 → 信息映射到契约维度 → 仅对缺口提问 → 按固定骨架生成 → 留痕时间戳。

## 边界

- 不限定项目类型，工具自判。
- knowledge 层只整理不逆推业务规则。
- 第一版不做平台 API 集成、不做静态站发布。
